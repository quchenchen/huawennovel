# 微短剧剧本创作 Skill v3.0

> 专为竖屏微短剧（9:16）设计的完整创作方法论与执行流程。
> 横屏思维是创作大忌。本 Skill 的所有规范均以竖屏为第一前提。

---

## 核心创作理论

**在执行任何命令之前，必须内化以下理论。它们不是参考，是铁律。**

### 一、竖屏视听语言铁律

#### 景别优先级

| 景别 | 适用性 | 核心用途 |
|------|--------|---------|
| 特写 CLOSE-UP | ✅ 首选 | 情绪爆发、眼神交锋、关键道具、手部动作 |
| 近景 MEDIUM CLOSE | ✅ 常用 | 对话、表情变化、人物反应 |
| 中景 MEDIUM SHOT | ✅ 常用 | 动作交代、双人对话、空间关系 |
| 全景 WIDE SHOT | ⚠️ 慎用 | 仅用于必要空间交代，全集占比不超过10% |
| 远景 EXTREME WIDE | ❌ 禁止 | 竖屏横向视野受限，远景信息量极低 |

**铁律：每个情绪高点必须用特写承载，不得用全景交代情绪。**

#### 构图规范

- **中心聚焦**：主体人物置于画面几何中心或黄金分割点，减少环境侵入
- **纵深构图**：用前景/背景的纵向层次制造空间感，替代横屏的横向纵深
- **人物充满画面**：重要情绪场景允许人物面部超出画框，强化沉浸感
- **禁止横向构图**：多人并排的横向构图在竖屏中效果极差，改用前后纵深排列

#### 场景描写对照表

| ❌ 横屏思维（禁止） | ✅ 竖屏写法（正确） |
|-------------------|-------------------|
| `△ （全景）大厅，灯火辉煌，人来人往。` | `△ 特写。旋转门玻璃反光里，她的侧脸一闪而过。` |
| `△ 远处，大厦出现在天际线上。` | `△ 近景。她抬起头，画外音是城市的喧嚣。` |
| `△ 两人站在停车场，保持距离。` | `△ 纵深构图。他在前景，她在景深处，两人之间是一段无法丈量的距离。` |
| `△ 会议室里，十几个人围坐一圈。` | `△ 特写切换。每张脸，每双眼睛，轮流出现在画面中央。` |

### 二、竖屏剧本创作方法论

#### 维度一：角色——绝对的主人公聚焦

- **核心人物上限 2 个**：配角只起穿针引线作用，不独立承担叙事线
- **双人戏黄金公式**：性格或身份有巨大反差的组合（强弱/阶层/立场对比），持续制造冲突
- **"面对面"交互感**：剧情张力由主角的表情、仪态、言语魅力承载，不依赖宏大叙事背景
- **小品化意识**：每集是一个独立的"影像小品"，主角卖力表演 > 精巧情节设计
- **人物标签三件套**（每个主要角色必须有，全剧保持一致）：
  - 标志性动作（如"指关节发白""放下茶杯慢半拍"）
  - 标志性台词模式（如"我知道了"——永远比别人先知道）
  - 标志性情绪反应（愤怒/恐惧/喜悦时的固定表现方式）

#### 维度二：节奏——"快讲"与"快切"协同

- **单集时长**：1-3分钟，去掉所有过渡性场景
- **爆点密度**：每15-30秒一个情感/剧情爆点（笑/泪/燃/爽任选其一）
- **高频翻转**：直接提供谜底揭晓的愉悦感，不设置需要观众推理的悬念
- **高潮前置**：高潮放在全集前2/3，最后1/3用于钩子和预告
- **正反切技术**：双人对话用A/B角交替正面向镜头，或上下分栏同框强调对立
- **前30秒法则**：每集开场前3段必须有冲突或悬念，禁止用纯环境描写开场
- **非线性工具**：关键反转集可用闪回（FLASHBACK）强化信息冲击

#### 维度三：台词——以语言为核心驱动

- **环境符号化**：场景只是表演空间的标签，不在环境描写上耗费笔墨
- **单次台词上限**：不超过3句，超过3句必须拆分为"动作+台词"交替
- **台词三特征**：
  - 快人快语：密集台词高效交代情节，背景信息通过对话传递，不用旁白
  - 唇枪舌剑：言语交锋本身就是动作，对话推进剧情
  - 夸张渲染：嵌入流行语、语气助词，增强日常气息与情绪张力
- **禁止内心独白**：不写"她心想……"，改为可见的动作或台词

#### 维度四：类型与受众——造梦逻辑

- **类型元素杂糅**：不必局限单一类型，可嫁接、戏仿熟悉桥段
- **情感逻辑驱动**：剧本是现代人的"白日梦"，情感共鸣 > 逻辑严密
- **受众认知**：小程序剧核心付费用户集中在35-50岁，选题娱乐化 ≠ 选题年轻化
- **动态调整**：可根据受众反馈随时修改后续剧本，试错成本低

---

### 三、单集结构标准

每集必须包含以下五个结构单元：

```
[开场钩子]  前30秒，制造冲突或悬念（禁止环境描写开场）
     ↓
[冲突升级]  核心矛盾爆发，每15-30秒一个爆点
     ↓
[高潮时刻]  全集前2/3处，情绪或剧情最高点，特写承载
     ↓
[余波处理]  高潮后的短暂喘息，为钩子蓄力
     ↓
[结尾钩子]  悬念切断，引导下集；💰集须制造强悬念
```

## 命令定义

### /start

**功能：** 选题定位，确定创作方向。

**流程：**
1. 加载 `genre-guide.md`，展示 13 种主流短剧题材（题材名、一句话描述、核心受众、典型爽点）
2. 用户选择题材（支持叠加）
3. 确认配置：
   - **目标受众：** 男频 / 女频 / 全年龄（提示：小程序剧付费主力为35-50岁）
   - **故事基调：** 爽燃 / 甜虐 / 搞笑 / 暗黑 / 温情
   - **结局类型：** 大团圆 / 开放式 / 反转式 / 悲剧
   - **集数规模：** 50-60集（紧凑）/ 60-80集（标准）/ 80-100集（长线）
   - **输出语言：** 中文（国内）/ English（出海，自动切换 /overseas）
4. 保存状态到 `.drama-state.json`

**输出：**
```
# 创作方向确认
- 题材组合：{题材}
- 目标受众：{受众}
- 故事基调：{基调}
- 结局类型：{结局}
- 集数规模：{集数}集
- 输出模式：{国内/overseas}

✅ 方向已锁定！输入 /plan 开始构建故事骨架
```

### /plan

**功能：** 生成完整的故事骨架和创作策略。

**前置条件：** 已完成 /start

**加载参考：** opening-rules.md, paywall-design.md, rhythm-curve.md, satisfaction-matrix.md

**生成内容：**
1. **剧名备选**（3个），每个附一句话说明
2. **时空背景**：时代、地点、社会环境、阶层关系
3. **一句话故事线** + **核心冲突**
4. **观众认知阶段设计**（竖屏剧核心结构）：
   - 将全剧分为3-5个阶段
   - 每个阶段：观众以为在看什么 / 实际在铺什么 / 结束时的认知跃迁
   - 标注集数范围
5. **三幕结构**（辅助框架）：建置/对抗/高潮结局的集数范围和核心事件
6. **全剧节奏波形**：高潮点、低谷点、付费卡点位置（文字描述）
7. **付费卡点规划**：具体集数 + 卡点类型（悬念钩/信息钩/情感钩）+ 悬念设计
8. **爽点矩阵**：按 satisfaction-matrix.md 规划全剧爽点分布
9. **反转节奏规划**：全剧反转总数、分布区间、每2-3集一个小反转的节点预设
10. **结局设计**：主线结局 + 感情线结局 + 伏笔回收

**输出：** 保存为 `creative-plan.md`

**结束提示：** `✅ 创作方案已保存！输入 /characters 开始塑造人物`

---

### /characters

**功能：** 生成完整角色体系。

**前置条件：** 已完成 /plan

**加载参考：** villain-design.md

**生成内容：**
1. **主要角色档案**（每个角色包含）：
   - 姓名、年龄、外貌特征（2-3句，侧重竖屏近景下的辨识度）
   - 性格关键词（3-5个）
   - 公开身份 vs 真实身份
   - 核心动机 / 最大冲突点 / 爽点功能
   - **人物标签三件套**（竖屏必须项）：
     - 标志性动作（每次出场必须出现一次）
     - 标志性台词模式
     - 标志性情绪反应（愤怒/恐惧/喜悦时的固定表现）

2. **角色关系图**（Mermaid 格式）

3. **角色弧线**：每个主要角色从第一集到最后一集的变化轨迹（集数标注）

4. **感情线弧线**：男女主关系发展的关键节点（集数标注）

5. **关键互动场景预设**：第一次冲突 / 身份揭露 / 感情转折 / 终极对决

6. **反派体系**（按 villain-design.md 的4层结构）

**输出：** 保存为 `characters.md`

**结束提示：** `✅ 角色档案已保存！输入 /outline 规划全剧分集`

---

### /outline

**功能：** 生成全剧分集目录。

**前置条件：** 已完成 /characters

**加载参考：** paywall-design.md, rhythm-curve.md

**每集条目格式：**
```
第{N}集：{集标题} —— {核心冲突或爽点一句话} {标记}
```

**标记：** 🔥 关键剧情集 / 💰 付费卡点集 / 无标记 = 常规推进集

**节奏要求：**
- 前10集：至少3个🔥，2个💰
- 全剧🔥占比：25-35%；💰占比：10-15%
- 每2-3集必须有一个反转或爆点
- 目录必须体现观众认知阶段的节奏变化

**输出：** 保存为 `episode-directory.md`

**结束提示：** `✅ 分集目录已保存！请通读目录确认节奏，然后输入 /episode 1 开始写第一集`

### /episode {N}

**功能：** 生成第 N 集的完整剧本。

**前置条件：** 已完成 /outline

**加载参考：** rhythm-curve.md, satisfaction-matrix.md, hook-design.md（第1集额外加载 opening-rules.md）

**支持格式：**
- `/episode 1` — 写第1集
- `/episode 5-8` — 批量写第5到第8集
- `/episode next` — 写下一集（自动递增）

**执行前必须确认：**
- 回顾前一集的结尾钩子，本集开场必须接住
- 确认本集在 episode-directory.md 中的定位（🔥/💰/常规）
- 确认本集涉及角色的标签三件套

**单集剧本格式（国内模式）：**

```markdown
# 第{N}集：{集标题}

> 本集定位：{🔥关键集 / 💰付费卡点集 / 常规推进集}
> 本集爽点：{爽点类型}
> 前情提要：{上一集结尾钩子，1句}
> 本集目标：{本集要完成的叙事任务，1句}

---

## 场次一

INT./EXT. {地点大写} - {时间}
出场：{人物}

△ {动作描述，竖屏视角，禁止全景开场}

**{角色名}**（{语气/动作}）："{台词，不超过3句}"

△ 特写。{情绪细节}

♪ {音乐氛围}

---

## 场次{N}（结尾场次）

△ 特写。{结尾情绪定格}

---

> 🎣 本集钩子：{悬念，切断在最高张力处}
> 📺 下集预告：{下集核心看点，1句}
```

**单集质量铁律（写完必须自查）：**

| 检查项 | 标准 |
|--------|------|
| 开场 | 前3段有冲突或悬念，无纯环境描写 |
| 爆点密度 | 每15-30秒一个爆点 |
| 高潮位置 | 在全集前2/3处 |
| 景别 | 特写/近景/中景为主，全景≤10%，无远景 |
| 构图 | 关键场景有构图说明 |
| 台词 | 单次不超过3句，无内心独白 |
| 角色标签 | 主要角色标志性动作/台词至少出现一次 |
| 钩子 | 结尾有悬念切断，💰集须强悬念 |

**输出：** 保存为 `episodes/ep{NNN}.md`（三位数补零）

**结束提示：** `✅ 第{N}集已保存！输入 /episode {N+1} 继续，或 /review {N} 检查质量`

### /review {N}

**功能：** 对已完成的剧本进行质量检查。

**支持格式：** `/review 5` / `/review 1-10` / `/review all`

**六维评分（每项10分，满分60分）：**

| 维度 | 检查内容 |
|------|---------|
| **节奏** | 开场是否有冲突、每15-30秒是否有爆点、高潮是否在前2/3、有无拖沓过渡 |
| **爽点** | 数量是否足够、强度是否达标、类型是否多样（笑/泪/燃/爽至少2种） |
| **台词** | 单次是否超3句、是否通过对话交代背景（无旁白）、角色语言区分度 |
| **角色聚焦** | 主线是否超2条、配角是否喧宾夺主、标签三件套是否出现、面对面交互感 |
| **竖屏视听** | 有无横屏思维（全景/远景滥用）、情绪高点是否用特写、关键场景有无构图说明 |
| **连贯性** | 与前后集是否矛盾、角色行为是否一致、伏笔是否延续 |

**输出格式：**
```markdown
# 质量自检报告 - 第{N}集

## 评分
| 维度 | 得分 | 说明 |
|------|------|------|
| 节奏 | {X}/10 | {说明} |
| 爽点 | {X}/10 | {说明} |
| 台词 | {X}/10 | {说明} |
| 角色聚焦 | {X}/10 | {说明} |
| 竖屏视听 | {X}/10 | {说明} |
| 连贯性 | {X}/10 | {说明} |
| **总分** | **{X}/60** | |

## 问题清单
1. 【严重/建议】{问题} → {修改方案}

## 修改优先级
{按严重程度排列}
```

**评分标准：** 54-60优秀 / 42-53良好 / 30-41及格 / 30以下重写

### /revise {N}

**功能：** 定点修改某集的指定问题，不整集重写。

**支持格式：**
- `/revise 5` — 修改第5集（先展示问题清单，再定点修改）
- `/revise 5 节奏` — 只修改第5集的节奏问题

**流程：**
1. 读取目标集剧本
2. 展示当前版本的问题清单（如已有 /review 报告则直接引用）
3. 用户确认修改范围
4. 定点修改，保留其余内容不变
5. 修改后自动运行 /review 验证

**输出：** 覆盖保存原文件，文件头注明修改版本号和修改内容

### /export

**功能：** 将完成的剧本导出为 `.md` + `.docx` 两种格式。

**前置条件：** 至少完成部分集数

**步骤一：生成 MD 汇总文件**

内容结构：
```markdown
# {剧名}

## 元信息
| 项目 | 内容 |
|------|------|
| 类型 | {题材组合} |
| 集数 | {已完成}/{总集数} |
| 单集时长 | 约1-3分钟 |
| 目标受众 | {受众} |
| 故事基调 | {基调} |
| 总字数 | {统计} |

## 故事梗概
{一句话故事线}
{三幕结构概述，3-5句}

## 主要角色
{角色简表，含标签三件套}

## 分集剧本
{所有已完成集数的完整内容}
```

保存为 `export/{剧名}-完整剧本.md`

**步骤二：生成 DOCX（使用 minimax-docx 技能）**

调用 minimax-docx 技能，用 `docx-js`（Node.js）生成标准剧本格式 Word 文档。

格式规范：
- 页面：A4，页边距上下 2.5cm、左右 2.5cm
- 正文字体：仿宋（FangSong），小四（12pt = 24 半磅）
- 行距：单倍（240 DXA）
- 段前段后：0
- 场景标题：黑体加粗，小三（15pt），段前 6pt
- 集标题：黑体加粗，三号（16pt），居中，段前 12pt
- 角色名：居中，加粗
- 对话：左右各缩进 720 DXA（约4字符）

## 创作原则

1. **竖屏优先**：所有场景描写、景别、构图必须符合竖屏视听语言铁律，横屏思维是创作大忌
2. **渐进式创作**：每步确认后才进入下一步
3. **上下文连贯**：写后续集数前必须回顾前集结尾钩子，确保接住
4. **角色标签稳定**：主要角色的标签三件套全剧保持一致，不得随意改变
5. **质量可控**：每集写完自查铁律清单，不满意就用 /revise 定点修改
6. **动态调整**：可根据用户反馈随时修改后续剧本，试错成本低
